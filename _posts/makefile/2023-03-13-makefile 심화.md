저번에 Makefile 튜토리얼에 대해서 공부했었는데, Makefile에서 더 심화적으로 공부해볼 수 있는 것들에 대해 작성해보려 한다.

## **_컴파일 옵션_**

- 컴파일은 소스 코드를 컴퓨터가 이해할 수 있는 어셈블리어로 변환하는 과정
- -c 옵션: `링크를 하지 않고` 컴파일해서 목적 파일 (Object File)을 생성
- -o 옵션: 생성된 아웃풋 파일의 이름을 지정
- -g 옵션: 디버그 정보
- -L 옵션: 헤더 파일의 디렉토리, 즉 -l로 지정된 라이브러리가 존재하는 위치 공개
- -l 옵션: 라이브러리 지정, 즉 특정한 라이브러리(.a)를 포함하도록 지정
- -I 옵션: 컴파일러가 #include가 작성된 라인에 있는 헤더파일의 위치를 찾을 수 있게 위치 공개
- -Wall: 모든 경고 표시
- Werror: 모든 경고를 에러로 처리

## **_Makefile 매크로_**

make -p를 통해 Makefil에서 미리 정의된 매크로를 확인할 수 있다.

| 매크로 이름 | 설명                   | 기본값 |
| ----------- | ---------------------- | ------ |
| AR          | 아카이브 관리 프로그램 | ar     |
| AS          | 어셈블러               | as     |
| CC          | C 컴파일러             | cc     |
| CXX         | C++ 컴파일러           | g++    |
| CPP         | C 전처리기             | cc -E  |
| LD          | 링크                   | ld     |
| ARFLAGS     | AR 플래그              | -      |
| ASFLAGS     | 어셈블러 플래그        | -      |
| CFLAGS      | C 컴파일러 플래그      | -      |
| CXXFLAGS    | C++ 컴파일러 플래그    | -      |
| CPPFLAGS    | C 전처리기 플래그      | -      |

make -p를 통해 확인할 수 없지만 make 내부적으로 정의되어 있는 자동 매크로도 있다.

| 매크로 | 기능                                                      |
| ------ | --------------------------------------------------------- |
| $?     | 현재 타겟보다 최근에 변경된 종속 항목 리스트              |
| $^     | 현재 타겟의 종속 항목 리스트                              |
| $@     | 현재 타겟의 이름                                          |
| $<     | 현재 타겟보다 최근에 변경된 종속 항목 리스트              |
| $\*    | 현재 타겟보다 최근에 변경된 종속 항목의 이름              |
| $%     | 현재 타깃이 라이브러리 모듈일 때, .o 파일에 대응되는 이름 |

## **_아카이브 생성_**

ar crus [라이브러리 이름][오브젝트 파일들]

- c 옵션: 아카이브 파일이 존재하지 않아도 경고 메시지를 출력하지 않는다.
- r 옵션: 새로운 오브젝트 파일이면 추가, 기존 파일이면 치환한다.
- u 옵션: 오브젝트 파일의 타임스탬프를 비교해 새로운 파일일 경우에만 치환한다.
- s 옵션: 아카이브 인덱스를 생성한다.

아카이브 인덱스를 생성하지 않으면, 링크 속도가 느려지고 시스템 환경에 따라서 에러가 발생할 수도 있다.

## **_Makefile 작성_**

```bash
CC = gcc
CFLAGS = -W -Wall
TARGET = test
OBJS = main.o file1.o file2.o
RM = rm -f

all : $(TARGET)

$(TARGET) : $(OBJS)
	$(CC) $(CFLAGS) -o $@ $^

%.o : %.c
	$(CC) $(CFLAGS) -c $< -o $@

clean :
	$(RM) $(TARGET) $(OBJS)

.PHONY : clean
```

Makefile을 더 깊게 공부해볼 수 있지만, 다음에는 Cmake에 대해서 공부해보려 한다:-)
